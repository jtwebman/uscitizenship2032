# Contributing to USCitizenship2032

Thank you for your interest in this project. This is meant to be a collaborative effort and we welcome contributions from people across the political spectrum.

## How to contribute

### Feedback on the amendment text

The amendment language is a draft. It needs testing from lawyers, policy experts, and regular citizens. If you see a problem with the wording, an edge case we have not thought of, or a better way to phrase something:

1. Open an issue using the "Amendment Feedback" template
2. Be specific about which section you are addressing
3. Explain the problem and suggest a fix if you have one

### Legal analysis

If you have a legal background and can identify constitutional issues, conflicts with existing law, or edge cases, we especially want to hear from you. Use the "Legal Analysis" issue template.

### Site improvements

For bug fixes, design improvements, or new features:

1. Fork the repository
2. Create a branch for your change
3. Make your changes
4. Run `npm run build` to make sure the site still builds
5. Open a pull request

### Translations

We would love to make this content available in other languages. If you can translate, open an issue to coordinate so we do not duplicate work.

## Guidelines

### Be respectful

This is a politically sensitive topic. People will disagree. That is expected and welcome. What is not welcome:

- Personal attacks
- Bad faith arguments designed to derail rather than improve
- Partisan framing that assumes the other side is acting in bad faith

### Keep it simple

The site is written at a level that a wide audience can understand. If you are contributing content, please:

- Use short sentences
- Avoid jargon unless it is a legal term that needs to be exact
- Use simple punctuation (periods and commas, not semicolons or em dashes)
- Write like you are explaining something to a smart friend, not writing a law review article

### Cite your sources

If you make a factual claim, link to the source. This project is built on verifiable data, not opinions.

## Pull request process

1. Keep pull requests focused on one thing
2. Describe what you changed and why
3. If you are changing amendment language, explain the legal or practical reason
4. All pull requests need at least one review before merging

## Code of conduct

Be a good person. Treat others the way you want to be treated. If someone is making the project worse instead of better, they will be asked to leave.

## Questions?

Open an issue with the "Question" label and we will do our best to answer.
